# Use Cases (Task Manager)

Цей документ описує основні сценарії використання (Use Cases) для REST API системи керування задачами згідно з вимогами Лабораторної роботи №1.

---

## 1. Реєстрація користувача

- **Актор:** Анонімний користувач
- **Передумови:** Користувач не авторизований у системі.
- **Основний сценарій:**
  1. Користувач відправляє POST-запит з `email` та `password`.
  2. Система валідує формат email.
  3. Система хешує пароль.
  4. Система створює новий запис користувача в базі даних.
  5. Система повертає HTTP статус `201 Created` та дані створеного користувача (без пароля).
- **Альтернативні сценарії / помилки:**
  - Невалідний формат `email` або закороткий пароль: система повертає `400 Bad Request`.
  - Користувач з таким `email` вже існує: система повертає `409 Conflict`.

## 2. Авторизація користувача (Вхід)

- **Актор:** Анонімний користувач
- **Передумови:** Користувач вже зареєстрований у системі.
- **Основний сценарій:**
  1. Користувач відправляє POST-запит з `email` та `password`.
  2. Система знаходить користувача за `email`.
  3. Система перевіряє відповідність пароля хешу в БД.
  4. Система генерує JWT-токен.
  5. Система повертає HTTP статус `200 OK` разом із JWT-токеном.
- **Альтернативні сценарії / помилки:**
  - Користувача не знайдено або пароль невірний: система повертає `401 Unauthorized`.

## 3. Створення задачі

- **Актор:** Авторизований користувач
- **Передумови:** Користувач успішно пройшов автентифікацію та має дійсний JWT-токен.
- **Основний сценарій:**
  1. Користувач відправляє POST-запит з даними задачі (`title`, `description`, `dueDate`).
  2. Система перевіряє наявність обов'язкового поля `title`.
  3. Система перевіряє інваріант: `dueDate` (якщо передано) не може бути в минулому.
  4. Система створює задачу в БД, автоматично призначаючи їй статус `PENDING` та прив'язуючи до `userId` з токена.
  5. Система повертає HTTP статус `201 Created` та створений об'єкт задачі.
- **Альтернативні сценарії / помилки:**
  - Запит без JWT-токена або токен недійсний: система повертає `401 Unauthorized`.
  - Відсутнє поле `title`: система повертає `400 Bad Request`.
  - Порушено інваріант (дата дедлайну в минулому): система повертає `400 Bad Request` з відповідним повідомленням.

## 4. Отримання списку задач

- **Актор:** Авторизований користувач
- **Передумови:** Користувач має дійсний JWT-токен.
- **Основний сценарій:**
  1. Користувач відправляє GET-запит на отримання задач.
  2. Система отримує `userId` з токена.
  3. Система витягує з БД усі задачі, що належать цьому `userId`, відсортовані за датою створення (від новіших до старіших).
  4. Система повертає HTTP статус `200 OK` та масив задач у форматі JSON.
- **Альтернативні сценарії / помилки:**
  - Неавторизований запит: `401 Unauthorized`.

## 5. Оновлення задачі (редагування або зміна статусу)

- **Актор:** Авторизований користувач
- **Передумови:** Користувач має дійсний JWT-токен, задача з вказаним ID існує і належить цьому користувачу.
- **Основний сценарій:**
  1. Користувач відправляє PUT або PATCH-запит з ID задачі та новими даними (наприклад, зміна `status` на `IN_PROGRESS` або `COMPLETED`).
  2. Система перевіряє, чи належить задача поточному користувачу.
  3. Система перевіряє інваріанти (наприклад, нова `dueDate` не в минулому).
  4. Система оновлює дані задачі в БД.
  5. Система повертає HTTP статус `200 OK` та оновлену задачу.
- **Альтернативні сценарії / помилки:**
  - Неавторизований запит: `401 Unauthorized`.
  - Задача не знайдена або належить іншому користувачу: `404 Not Found`.
  - Невалідні дані для оновлення (дата в минулому, невідомий статус): `400 Bad Request`.

## 6. Видалення задачі

- **Актор:** Авторизований користувач
- **Передумови:** Користувач має дійсний JWT-токен, задача з вказаним ID існує і належить йому.
- **Основний сценарій:**
  1. Користувач відправляє DELETE-запит з ID задачі.
  2. Система перевіряє, чи існує задача і чи належить вона користувачу.
  3. Система видаляє задачу з БД.
  4. Система повертає HTTP статус `204 No Content`.
- **Альтернативні сценарії / помилки:**
  - Неавторизований запит: `401 Unauthorized`.
  - Задача не знайдена або належить іншому користувачу: `404 Not Found`.
